## 🚀 Быстрый старт

### 1. Клонируй и установи зависимости

```bash
git clone <repo>
cd nft_market_bot
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Настрой окружение

```bash
cp .env.example .env
nano .env   # заполни BOT_TOKEN и ADMIN_IDS
```

### 3. Запусти

```bash
python bot.py
```

База данных `nft_market.db` создастся автоматически при первом запуске.

## 🎯 Функциональность MVP

### 🛍 Маркет (фиксированная цена)
- Листинг с описанием, редкостью, ценой
- Пагинация, счётчик просмотров
- Покупка с мгновенным переводом подарка

### 🔨 Аукционы
- Старт, мин. шаг, время окончания
- Опциональный buyout (мгновенная покупка)
- Автозакрытие через планировщик (каждую минуту)
- Уведомления победителю и продавцу

### 💼 Портфолио
- Все NFT-подарки пользователя
- Детали по редкости (Common / Rare / Epic / Legendary)
- История сделок (покупки и продажи)
- Быстрый переход к продаже из портфолио

### 👥 Реферальная система
- Уникальная deep-link ссылка для каждого
- 1% от каждой продажи реферала — навсегда
- Статистика: кол-во рефералов, общий заработок

### 🔧 Админ-панель (`/admin`)
- Статистика платформы (пользователи, объём, комиссии)
- Бан пользователей
- Выдача NFT-подарков вручную

## 🗄️ База данных

По умолчанию: **SQLite** (файл `nft_market.db`) — идеально для старта.

### Миграция на PostgreSQL

1. Установи asyncpg: `pip install asyncpg`
2. В `.env` измени: `DATABASE_URL=postgresql+asyncpg://user:pass@host/dbname`
3. В `db/database.py` замени `aiosqlite.connect` на asyncpg-пул

Схема полностью совместима с PostgreSQL (чистый SQL без специфики SQLite).

## 💳 Интеграция реальных платежей TON

Сейчас бот работает с **внутренним балансом** (демо-режим).  
Для реальных платежей подключи:

1. **TON Connect** — [tonconnect.dev](https://tonconnect.dev/)
2. **Toncenter API** — для верификации on-chain транзакций
3. **Escrow-логика** — блокировка средств при ставке на аукцион

Рекомендуемый флоу:
```
Пользователь → TON Connect → твой кошелёк → бот верифицирует tx_hash → начисляет баланс
```

## 🔐 Безопасность

- Rate limiting: 0.5 сек между сообщениями, 0.3 сек между callback
- Проверка владельца при снятии с продажи / отмене аукциона
- Валидация всех числовых вводов
- Отдельный список `ADMIN_IDS` для защищённых команд
- `PRAGMA foreign_keys=ON` — целостность данных на уровне БД
